# Upload Results And Violation Surface Rules

Violation result assembly must remain deterministic and evidence-bound.

Do not generate speculative violations.

Every violation surfaced here must:
- map to evidence
- map to regulation
- map to canonical extracted data
- preserve bureau, creditor, and collector responsibility boundaries

No AI-generated violation may become active truth without deterministic validation or admin approval.

Preserve:
- violation category stability
- parser quality review signals
- artifact ownership checks
- severity and actionability semantics
- dispute packet readiness expectations
- existing response shape unless explicitly approved

Before changing this subsystem, trace effects into violation detection, evidence links, regulation references, admin correction, and dispute packet generation.

If a requested change expands violation truth, regulation mapping, or dispute packet behavior beyond the target scope, stop and produce an implementation plan and risk analysis before editing code.
